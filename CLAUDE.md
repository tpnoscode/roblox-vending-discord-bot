# Discord 자판기 봇 개발 가이드라인

이 프로젝트는 **Node.js**와 **discord.js**를 사용하는 Discord 자판기 봇 프로젝트의 초기 세팅 저장소입니다. 현재 단계의 목적은 실제 기능 구현이 아니라, Claude Code, Google Antigravity, GitHub 기반 협업과 배포 준비를 빠르게 시작할 수 있는 기본 구조를 마련하는 것입니다.

## 개발 우선순위

개발 도구는 기능 구현 시 다음 우선순위를 따릅니다.

| 우선순위 | 기준 | 설명 |
| --- | --- | --- |
| 1 | 실행 가능성 | 로컬과 배포 환경에서 바로 실행 가능한 구조를 유지합니다. |
| 2 | 관리 편의성 | 디스코드 슬래시 명령어 중심으로 운영자가 재고, 상품, 랜덤박스를 관리할 수 있게 확장합니다. |
| 3 | 자동화 | 24시간 실행, 자동 충전, 웹훅 수신, 재시작 자동화가 가능한 구조로 확장합니다. |
| 4 | 모듈화 | 상품, 재고, 충전, 랜덤박스, 로그, 관리자 권한 로직을 독립 모듈로 분리합니다. |

## 프로젝트 구조

| 경로 | 목적 |
| --- | --- |
| `src/index.js` | 봇 실행 진입점입니다. 현재는 환경 변수 확인용 최소 스텁만 둡니다. |
| `src/commands/` | 디스코드 슬래시 명령어 파일을 추가할 위치입니다. |
| `src/events/` | Discord 이벤트 핸들러 파일을 추가할 위치입니다. |
| `src/utils/` | 공통 유틸리티, 데이터베이스 연결, 외부 API 호출 함수를 둘 위치입니다. |
| `src/models/` | 상품, 재고, 주문, 충전, 랜덤박스 등 데이터 모델을 둘 위치입니다. |
| `config/` | 환경 설정 관련 파일을 둘 위치입니다. 실제 비밀값은 `.env`에서 관리합니다. |
| `docs/` | 개발 지시서, 연결 체크리스트, 배포 준비 문서를 보관합니다. |

## 코드 스타일

ESLint와 Prettier를 사용합니다. 변수명은 `camelCase`, 상수명은 `SCREAMING_SNAKE_CASE`를 사용합니다. 함수는 한 가지 목적만 갖도록 작성하고, 명령어 처리 로직과 데이터 처리 로직을 분리합니다.

## Claude Code 작업 방식

새 기능을 구현할 때는 먼저 파일 경로와 목표를 명확히 지정합니다. 예를 들어 `src/commands/inventory/addStock.js`에 `/재고추가` 명령어 스켈레톤을 만들고, 데이터 저장 방식은 별도 모듈로 분리하라고 지시합니다.

복잡한 기능은 한 번에 완성하려 하지 말고, **명령어 등록 → 데이터 저장 → 관리자 확인 → 디스코드 응답 → 배포 확인** 순서로 나누어 작업합니다.

## Google Antigravity 작업 방식

Antigravity에는 이 저장소를 연결한 뒤, `docs/ANTIGRAVITY_PROMPT.md`를 첫 지시문으로 사용합니다. Antigravity가 코드를 생성할 때도 이 파일의 구조를 유지하고, 모든 변경 사항은 GitHub에 커밋되도록 진행합니다.

## 기능 구현 금지선

현재 저장소는 **초기 세팅용**입니다. 다음 항목은 아직 구현하지 않습니다.

| 항목 | 현재 처리 |
| --- | --- |
| 실제 상품 지급 로직 | 추후 상세 요구사항 확정 후 구현합니다. |
| 자동 충전 처리 로직 | 결제/입금 방식 확정 후 구현합니다. |
| 랜덤박스 확률 로직 | 상품 테이블과 확률 정책 확정 후 구현합니다. |
| 관리자 전용 명령어 전체 구현 | 명령어 목록 확정 후 구현합니다. |

## 실행 기준

`.env.example`을 복사해 `.env`를 만들고 필요한 값을 채운 뒤, `npm install`과 `npm run dev`로 기본 실행 상태를 확인합니다. 실제 봇 로그인은 `src/index.js`에서 토큰 연결 코드를 추가한 뒤 진행합니다.
